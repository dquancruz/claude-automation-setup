## 🎯 PULL REQUEST ESSENTIALS

### PR Title Format

Debe ser **idéntico** al commit message principal:

```
[EMOJI] [TYPE] | [DESCRIPCIÓN] [TICKET]

Ejemplos:
✨ Feature | Add date filter [PROJ-123]
🐛 Fix | Handle null dates [PROJ-124]
♻️ Refactor | Extract validation [PROJ-125]
📚 Docs | Update API docs [PROJ-126]
```

### PR Description Template

Generado automáticamente por pr-manager skill:

```markdown
## What
- Changed X
- Added Y
- Fixed Z

## Why
Problem statement and business value

## Testing
- Unit tests: 8/8 passing ✅
- Integration: 5/5 passing ✅
- Coverage: 97%

## Related
- Jira: [PROJ-123](...)
- Design: [Figma](...)
```

### PR Metadata

- **Title:** [EMOJI] [TYPE] | [DESC] [TICKET]
- **Description:** Auto-generated from PR-Description-Formatter skill
- **Labels:** auto-added (enhancement, bug, documentation, etc)
- **Assignees:** Optional (assign reviewers)
- **Reviewers:** auto-assigned (code-reviewer-pro, team lead)
- **Branch:** feature/PROJ-123-description
- **Base:** main
- **Compare:** feature/PROJ-123-description ... main

## 🔍 VALIDATIONS BEFORE PR CREATION

Script debe validar:

### 1. Branch exists and has commits

```bash
# Get current branch
BRANCH=$(git rev-parse --abbrev-ref HEAD)

# Verify not on main/master
if [ "$BRANCH" = "main" ] || [ "$BRANCH" = "master" ]; then
  echo "❌ Cannot create PR from main/master"
  exit 1
fi

# Verify commits exist
COMMITS=$(git log main..$BRANCH --oneline | wc -l)
if [ $COMMITS -eq 0 ]; then
  echo "❌ No commits in branch (already merged?)"
  exit 1
fi
```

### 2. No conflicts with main

```bash
# Check for merge conflicts
git fetch origin main
if ! git merge-base --is-ancestor main $BRANCH; then
  # Branch is ahead of main (good)
  if git merge --no-commit --no-ff origin/main $BRANCH 2>/dev/null; then
    git merge --abort
    echo "✅ No conflicts with main"
  else
    echo "❌ Merge conflicts with main"
    exit 1
  fi
fi
```

### 3. Tests passed in last commit

```bash
# Get latest commit SHA
SHA=$(git rev-parse HEAD)

# Check GitHub Actions status
# (via GitHub API)
STATUS=$(curl -s https://api.github.com/repos/ORG/REPO/commits/$SHA/status \
  | jq -r '.state')

if [ "$STATUS" != "success" ]; then
  echo "❌ Tests not passing (status: $STATUS)"
  exit 1
fi
```

### 4. Code review passed

```bash
# Check if approved by code-reviewer-pro
# (via code review logic)

if ! code_review_passed(); then
  echo "❌ Code review not approved"
  exit 1
fi
```

## 📋 PR CREATION WORKFLOW

### Step 1: Gather info

```javascript
const prInfo = {
  title: "✨ Feature | Add date filter [PROJ-123]",
  body: "<auto-generated from pr-manager>",
  head: "feature/PROJ-123-date-filter",  // feature branch
  base: "main",                          // base branch
  draft: false,
  labels: ["enhancement", "jira"],
  reviewers: ["code-reviewer-pro", "tech-lead"],
  assignees: ["author@company.com"]     // optional
};
```

### Step 2: Validate

```javascript
// 1. Branch exists with commits
// 2. No conflicts with main
// 3. Tests passing
// 4. Code review approved
// 5. Description ready
```

### Step 3: Create PR via GitHub API

```javascript
const octokit = new Octokit({
  auth: process.env.GITHUB_TOKEN
});

const pr = await octokit.pulls.create({
  owner: 'org',
  repo: 'repo',
  title: prInfo.title,
  body: prInfo.body,
  head: prInfo.head,
  base: prInfo.base,
  draft: prInfo.draft
});

console.log(`✅ PR created: #${pr.data.number}`);
console.log(`URL: ${pr.data.html_url}`);
```

### Step 4: Add metadata

```javascript
// Add labels
await octokit.issues.addLabels({
  owner: 'org',
  repo: 'repo',
  issue_number: pr.data.number,
  labels: prInfo.labels
});

// Add reviewers
await octokit.pulls.requestReviewers({
  owner: 'org',
  repo: 'repo',
  pull_number: pr.data.number,
  reviewers: prInfo.reviewers
});

// Add assignees (if any)
if (prInfo.assignees.length > 0) {
  await octokit.issues.addAssignees({
    owner: 'org',
    repo: 'repo',
    issue_number: pr.data.number,
    assignees: prInfo.assignees
  });
}
```

### Step 5: Report status

```
✅ PR CREATED

PR #456
Title: ✨ Feature | Add date filter [PROJ-123]
URL: https://github.com/org/repo/pull/456

Description: ✅ Auto-generated
Labels: enhancement, jira ✅
Reviewers: code-reviewer-pro ✅
Tests: Running... (GitHub Actions)

Next: Waiting for code review ⏳
```

## ⚠️ ERROR HANDLING

### Error: Branch not found

```
❌ Error: Branch 'feature/PROJ-123' not found

Fix:
1. Verify branch exists: git branch -a
2. Fetch latest: git fetch origin
3. Verify commits: git log main..BRANCH
```

### Error: Conflicts with main

```
❌ Error: Merge conflicts with main

Fix:
1. Resolve conflicts locally:
   git fetch origin main
   git merge origin/main
   (fix conflicts)
   git add .
   git commit -m "Resolve conflicts"
2. Retry PR creation
```

### Error: Tests not passing

```
❌ Error: Tests not passing (status: pending)

Fix:
1. Wait for GitHub Actions to complete
2. Check Actions tab for failures
3. Fix failures locally:
   npm test
   (fix failures)
   git add .
   git commit -m "fix(test): resolve test failures"
   git push origin BRANCH
4. Retry PR creation once tests pass
```

### Error: Missing code review

```
❌ Error: Code review not approved

Fix:
1. Run code-reviewer-pro locally
2. Fix issues found
3. Retry PR creation
```
